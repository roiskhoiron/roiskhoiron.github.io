# Flutter Null Safety

Null Safety adalah fitur paling penting di Dart modern. Dengan Null Safety, kode kamu jadi **lebih aman** dan **lebih mudah diprediksi**.

## Apa itu Null Safety?

Sebelum Null Safety, semua variabel di Dart bisa bernilai `null`. Masalahnya? Satu lupa ngecek `null` bisa bikin aplikasi **crash** tiba-tiba.

```dart
// Tanpa Null Safety — bisa crash!
String name = getName();
print(name.length); // ERROR kalau name == null
```

Dengan Null Safety, Dart **memaksa** kamu buat ngasih tau: "variabel ini boleh null apa enggak".

```dart
// Dengan Null Safety
String name = getName();    // WAJIB ada nilainya
String? maybeName = null;  // BOLEH null (tanda ?)
```

## Cara Kerja Null Safety

### 1. Tipe biasa (`String`, `int`) — **TIDAK** boleh null
```dart
String name = "CodingSkuy";
name = null; // ERROR! ❌
```

### 2. Tipe nullable (`String?`, `int?`) — BOLEH null
```dart
String? name = null; // OK ✅
name = "CodingSkuy";  // OK ✅
```

### 3. Akses properti nullable
```dart
String? name = getName();
print(name?.length); // Aman — kalau null, hasilnya null
print(name!.length); // Paksa — hati-hati! CRASH kalau null
```

## Kenapa Penting buat Flutter?

Flutter aplikasi kamu bakal:

- ✅ **Lebih stabil** — gak ada NullPointerException dadakan
- ✅ **Kode lebih bersih** — gak perlu ribet ngecek `if (x != null)` terus
- ✅ **Performanya sama** — gak ada overhead runtime
- ✅ **Auto-detect error** — IDE langsung kasih tau kalau ada yang salah

## Praktek Singkat

Coba kode ini di editor Flutter kamu:

```dart
void main() {
  // 1. Tanpa nullable — WAJIB diisi
  String message = "Halo CodingSkuy!";
  print(message.length); // 16 ✅

  // 2. Nullable — BOLEH null
  String? name;
  print(name?.length); // null (gak crash) ✅

  // 3. Null-aware operator
  String displayName = name ?? "Teman CodingSkuy";
  print(displayName); // "Teman CodingSkuy" ✅
}
```
