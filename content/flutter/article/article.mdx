# Flutter Null Safety

> **Kamu pernah nemu error `NullPointerException` pas aplikasi tiba-tiba crash?** Tenang, kamu gak sendirian. Di artikel ini kita bakal bahas gimana Null Safety di Dart nyelesain masalah ini.

---

## Masalahnya

Bayangin kode ini:

```dart
String nama = null; // Eits, bakal error!
print(nama.length);
```

Di Dart versi lama, kode di atas bakal **crash** pas dijalanin. Kenapa? Karena `nama` itu `null`, tapi kita maksa ambil `.length`-nya. Jadilah **NullPointerException** — musuh besar para developer.

> **Null Safety** hadir buat mencegah error kayak gini SEBELUM aplikasi jalan. Bukan pas udah jalan terus crash.

---

## Kenapa Ini Penting?

Tanpa Null Safety:

| Masalah | Dampak |
|---|---|
| Error baru ketahuan pas runtime | Aplikasi crash di tangan user |
| Developer harus ingat cek null manual | Lupa → crash |
| Kode penuh `if (x != null)` | Bikin kode berantakan |

Dengan Null Safety:

| Keuntungan | Dampak |
|---|---|
| Error ketahuan pas compile | Aman sebelum dijalankan |
| Tipe data jelas: `String` vs `String?` | Tau mana yang bisa null |
| Kode lebih bersih | Gak perlu cek null manual |

---

## Konsep Inti

### 1. Type Safety — `String` vs `String?`

Di Dart dengan Null Safety:

| Tipe | Bisa null? | Contoh |
|---|---|---|
| `String` | ❌ Tidak | `String nama = 'Budi'` |
| `String?` | ✅ Iya | `String? nama = null` |

```dart
// Tanpa Null Safety — bahaya!
String nama = null; // Error pas runtime (di Dart lama)

// Dengan Null Safety
String? nama = null; // ✅ Aman — Dart tau ini nullable
```

### 2. Cara Aman Akses Nullable

Ada 3 cara utama:

```dart
String? nama;

// Cara 1: null-aware operator (?.)
print(nama?.length); // Output: null (gak crash!)

// Cara 2: null-coalescing (??)
print(nama?.length ?? 0); // Output: 0 (default kalau null)

// Cara 3: assertion (!) — hati-hati!
print(nama!.length); // Bakal crash kalau nama null!
```

### 3. Late Initialization — `late`

Kalau kamu yakin variabel bakal diisi SEBELUM dipake, tapi gak bisa diisi pas deklarasi:

```dart
late String nama;

void main() {
  nama = 'Budi';
  print(nama.length); // ✅ Aman — udah diisi sebelumnya
}
```

**Peringatan:** Kalau `late` variabel dipake SEBELUM diisi, tetap crash!

---

## Contoh Lengkap

```dart
// Dart dengan Null Safety

class User {
  final String nama; // Wajib diisi, gak boleh null
  final String? email; // Boleh null

  User({required this.nama, this.email});

  String get displayInfo {
    // null-aware: kalau email null, tampilkan pesan
    final emailDisplay = email ?? '(tidak ada email)';
    return '$nama — $emailDisplay';
  }
}

void main() {
  final user1 = User(nama: 'Budi', email: 'budi@email.com');
  final user2 = User(nama: 'Siti'); // email: null

  print(user1.displayInfo); // Budi — budi@email.com
  print(user2.displayInfo); // Siti — (tidak ada email)

  // Aman: null-aware access
  print(user1.email?.length); // 15
  print(user2.email?.length); // null
}
```

## Common Mistakes

### ❌ Salah: Pake `!` sembarangan

```dart
String? nama;
print(nama!.length); // 💥 Crash! nama masih null
```

**✅ Bener:** Pake `?.` atau `??`

```dart
print(nama?.length ?? 0); // Aman
```

### ❌ Salah: Lupa `late` bisa crash

```dart
late int umur;
print(umur); // 💥 Crash! umur belum diisi
```

**✅ Bener:** Pastikan diisi sebelum dipake

```dart
late int umur;
umur = 17;
print(umur); // ✅ Aman
```

### ❌ Salah: Anggep `String?` sama dengan `String`

```dart
String? nama;
String pastiString = nama; // 💥 Error compile!
```

**✅ Bener:** Handle null-nya dulu

```dart
String pastiString = nama ?? 'default';
```
