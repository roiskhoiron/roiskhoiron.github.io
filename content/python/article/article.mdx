# Python Developer

> **Kamu udah bisa Python dasar tapi masih bingung gimana jadi developer beneran?** Virtual environment, typing, pip, testing — tenang, kita bahas semua.

---

## Masalahnya

Coba kode ini:

```python
import requests

response = requests.get('https://api.example.com')
print(response.json())
```

**Error:** `ModuleNotFoundError: No module named 'requests'`

Tanpa environment management:

| Masalah | Dampak |
|---|---|
| Package bentrok antar project | "Butuh pandas v1 tapi v2 terinstall" |
| Salah versi Python | Script gak bisa jalan di server |
| Gak tracking dependensi | Lupa package apa yang dipake |

**Python Developer** = tau cara ngatur environment, bikin kode yang rapi, dan pake tools yang bener.

---

## Kenapa Ini Penting?

| Skill | Manfaat |
|---|---|
| Virtual Environment | Project aman — gak saling ganggu |
| Type Hints | Kode lebih jelas, error ketahuan sebelum runtime |
| Package Management | Version lock — reproducible |
| Testing | Kode gak gampang rusak pas diubah |

Tanpa ini? Kamu developer yang **hanya bisa nulis script**, bukan developer yang **bikin sistem**.

## Konsep Inti

### 1. Virtual Environment

```bash
# Bikin environment
python -m venv .venv

# Aktifin (Mac/Linux)
source .venv/bin/activate

# Aktifin (Windows)
.venv\Scripts\activate

# Matiin
deactivate

# Simpan dependensi
pip freeze > requirements.txt

# Install dari file
pip install -r requirements.txt
```

### 2. Type Hints

```python
# Tanpa type — gak jelas
def sapa(nama):
    return f"Halo, {nama}"

# Dengan type — jelas input/output
def sapa(nama: str) -> str:
    return f"Halo, {nama}"

# Type untuk list & dict
from typing import List, Dict, Optional

users: List[Dict[str, any]] = [
    {"nama": "Budi", "umur": 17}
]

# Optional — bisa None
def cari_user(id: int) -> Optional[dict]:
    return None  # valid
```

### 3. Struktur Project

```
project/
├── src/
│   ├── __init__.py
│   └── main.py
├── tests/
│   ├── __init__.py
│   └── test_main.py
├── .venv/
├── requirements.txt
├── .gitignore
└── README.md
```

## Contoh Lengkap

```python
# src/calculator.py
from typing import Union

Number = Union[int, float]

def tambah(a: Number, b: Number) -> Number:
    """Menambahkan dua angka."""
    return a + b

def bagi(a: Number, b: Number) -> Number:
    """Membagi dua angka."""
    if b == 0:
        raise ValueError("Pembagi tidak boleh 0")
    return a / b

def main() -> None:
    print(tambah(10, 5))     # 15
    print(bagi(10, 2))       # 5.0

if __name__ == "__main__":
    main()
```

```python
# tests/test_calculator.py
import pytest
from src.calculator import tambah, bagi

def test_tambah():
    assert tambah(2, 3) == 5
    assert tambah(-1, 1) == 0

def test_bagi():
    assert bagi(10, 2) == 5.0

def test_bagi_nol():
    with pytest.raises(ValueError):
        bagi(10, 0)
```

## Common Mistakes

### ❌ Salah: Install package **global**

```bash
pip install flask  # tanpa venv → global
```

**✅ Bener:**

```bash
python -m venv .venv
source .venv/bin/activate
pip install flask
```

### ❌ Salah: Gak pake type hints

```python
def proses(data):
    # Apa itu `data`? List? Dict? String?
```

**✅ Bener:**

```python
def proses(data: List[dict]) -> bool:
```

### ❌ Salah: Commit `.venv` ke git

```bash
git add .venv  # ❌ besar & gak perlu
```

**✅ Bener:** Tambah `.venv` ke `.gitignore`

### ❌ Salah: Gak ada `requirements.txt`

Project orang lain: "Package apa yang dipake?!"

**✅ Bener:** `pip freeze > requirements.txt`
