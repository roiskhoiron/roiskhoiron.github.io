# Hive vs SQLite

> *"Hive itu cepat. Tapi pas butuh JOIN — SQLite lah jawabannya."*

Dua opsi database lokal Flutter. Sama-sama populer, beda use case.

**Hive:**
- Key-value NoSQL
- Super cepat baca/tulis
- Tidak support query kompleks
- Tidak ada relasi

**SQLite (via sqflite/drift):**
- Relational SQL
- Support JOIN, filter, aggregate
- Lebih lambat dari Hive untuk operasi sederhana
- Butuh schema definition

**Ceritaku:**
Awalnya pake Hive untuk cache daftar produk. Cepet, gampang. Tapi begitu perlu filter "produk dengan harga >50000 diurutkan berdasarkan rating" — Hive harus load semua data, filter manual di kode.

Migrasi sebagian ke SQLite untuk data yang butuh query kompleks. Hive tetap untuk cache session dan preference.

> *"Hive untuk cache dan data sederhana. SQLite untuk data relasional yang butuh query. Keduanya bisa dipakai bersamaan."*
