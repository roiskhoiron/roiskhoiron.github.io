# Monolith vs Modular

> *"Semua di satu folder itu cepat — sampai kamu harus cari di 200 file untuk satu bug."*

Dulu struktur proyek Flutter-ku rata:

```
lib/
├── models/
├── views/
├── controllers/  
├── services/
├── widgets/
└── utils/
```

Awalnya rapi. Tapi setelah 20 fitur, folder `views/` berisi 40 file. Mau hapus fitur profil? Harus cari di 7 folder berbeda.

**Modular approach:**

```
lib/
├── core/            — shared: theme, network, routing
├── features/
│   ├── auth/        — login, register, forgot password
│   ├── profile/     — lihat & edit profil
│   ├── dashboard/   — home screen
│   └── settings/    — pengaturan
└── shared/          — reusable widgets
```

**Kapan monolith:**
- App kecil (&lt;10 screen)
- Tim sendiri
- Belum tahu persis boundary fitur

**Kapan modular:**
- App makin besar
- Tim >2 orang
- Fitur bisa dihapus/ditambah independen

> *"Modular tidak harus modular di awal. Refactor ke modular saat merasa 'kok susah nyari file ini?'"*
