# Backend Engineer

> *"User tidak pernah lihat backend. Tapi kalau backend down, semua tahu."*

Backend engineer ada di belakang layar. Tidak kelihatan. Tapi tanpanya, aplikasi cuma UI cantik tanpa data.

**Perjalanan:**
Dari Laravel di Dicoding — bikin API untuk aplikasi learning management. Dari situ aku paham: backend bukan sekadar "return JSON".

**Skill inti:**

1. **API design** — REST atau GraphQL, versioning, error handling.
2. **Database** — schema design, indexing, query optimization.
3. **Security** — authentication, authorization, rate limiting, input validation.
4. **Infrastructure** — deployment, scaling, monitoring.

**Pelajaran paling berharga:**

Backend yang baik tidak terlihat. Kalau user komplen "aplikasi lambat" — 90% penyebabnya ada di backend. Kalau user komplen "error terus" — backend juga.

> *"Backend engineer adalah yang memastikan user tidak pernah mikir: 'kok lambat sih?'"*
