# Project Presentation

> *"Fitur keren tidak berguna kalau kamu tidak bisa menjelaskannya."*

Pernah lihat presentasi proyek yang membosankan? Slide penuh teks, presenter baca, audiens ngantuk.

Aku juga pernah begitu.

**Cara presentasi proyek yang efektif:**

1. **Cerita, bukan fitur** — jangan mulai dengan "aplikasi ini pake Flutter, Riverpod, Firebase." Mulai dengan "di sini ada masalah: user susah cari informasi coding dalam Bahasa Indonesia."

2. **Masalah → solusi → impact** — 3 slide aja cukup. Slide 1: masalah. Slide 2: solusimu. Slide 3: dampaknya.

3. **Demo yang terstruktur** — jangan asal klik. Ceritain: "sekarang saya akan login, lalu masuk ke halaman utama, dan menunjukkan fitur pencarian." Kalau demo error — jelaskan kenapa, itu malah menunjukkan pengetahuanmu.

4. **Kenali audiens** — presentasi ke engineer lain? Bahas teknis. Ke recruiter? Bahas impact. Ke investor? Bahas market.

> *"Presentasi proyek yang baik membuat audiens berpikir: 'saya mau pake/ hiring/ investasi orang ini'."*
