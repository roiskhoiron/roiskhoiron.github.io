# GitHub

> *"Dulu GitHub-ku cuma fork repositori orang. Sekarang jadi tempat orang lihat kualitas coding-ku."*

GitHub adalah portfolio teknis paling autentik. Tidak bisa bohong — kode berbicara.

**Optimasi GitHub:**

1. **Profile README** — repositori khusus dengan username yang sama. Isi: perkenalan singkat, tech stack utama, pinned repositori.
2. **Pinned repositori** — pilih 3-6 repositori terbaik. Bukan fork latihan. Tapi proyek yang kamu bangun.
3. **Readme per repositori** — jelaskan: apa ini? cara run? tech stack? screenshot.
4. **Commit message yang rapi** — bukan "fix bug" atau "update."
5. **Kontribusi aktif** — green square itu sinyal: orang ini aktif ngoding.

**Yang sering dilupakan:**

- Repositori privat tidak terlihat. Pastikan ada repositori publik.
- Fork tanpa kontribusi tidak berarti. Fork trus commit — baru bermakna.
- Dokumentasi di repositori sama pentingnya dengan kode.

> *"GitHub yang baik membuat rekruter tidak perlu tanya: 'bisa ngoding?'"*
