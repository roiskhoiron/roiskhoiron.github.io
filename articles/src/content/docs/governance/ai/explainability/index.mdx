# Explainability

> *"Boss bertanya: 'Kenapa model ini menolak pengajuan si A?' Aku cuma bisa jawab: 'Soalnya model bilang gitu.' Jawaban yang memalukan."*

Model AI sering disebut black box. Kita tahu input dan output, tapi tidak tahu apa yang terjadi di dalam. Selama model cuma buat rekomendasi film — tidak masalah. Tapi untuk keputusan yang impact-nya besar — masalah.

---

### Kenapa Explainability Penting?

**Kepercayaan** — pengguna lebih percaya jika tahu alasannya. "Rekomendasi ini karena kamu suka X." Lebih baik dari "Ini rekomendasi buat kamu."

**Debugging** — waktu model error, explainability bantu cari penyebabnya. Kenapa model salah klasifikasi? Fitur mana yang membuatnya salah?

**Compliance** — beberapa regulasi minta penjelasan. GDPR punya "right to explanation." Kamu harus bisa jelasin kenapa algoritma ambil keputusan tertentu.

**Fairness** — explainability bantu deteksi bias. Kalau fitur "gender" berkontribusi besar pada keputusan — ada masalah.

---

### Tingkat Explainability

**Global** — memahami model secara keseluruhan. Fitur apa yang paling berpengaruh? Secara umum, bagaimana model bekerja?

**Local** — memahami satu prediksi spesifik. Kenapa untuk user A hasilnya ditolak, untuk user B diterima?

---

### Metode yang Kupakai

**SHAP** — untuk model tabular. Kasih tahu kontribusi setiap fitur terhadap prediksi. Contoh: "Fitur 'pendapatan' berkontribusi 60%, 'usia' 30%, 'lokasi' 10%."

Paling sering kupakai karena implementasinya tidak terlalu rumit.

**LIME** — untuk teks/gambar. Bikin model sederhana di sekitar satu prediksi. Cocok untuk explainability lokal.

---

### Trade-off

Model kompleks → akurasi tinggi → susah dijelaskan.
Model sederhana → akurasi lebih rendah → mudah dijelaskan.

Pilih sesuai kebutuhan. Untuk aplikasi medis, explainability mungkin lebih penting daripada 1% akurasi. Untuk recommendation system, explainability tidak terlalu kritis.

> *"Kalau modelmu tidak bisa menjelaskan keputusannya, apakah kamu percaya modelmu?"*
