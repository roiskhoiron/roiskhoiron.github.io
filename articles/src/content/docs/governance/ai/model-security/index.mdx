# Model Security

> *"Model AI itu kayak resep rahasia. Kalau resepnya bocor, kompetitor bisa bikin versi sendiri."*

Aku pernah bikin model yang cukup akurat — investasi waktu dan komputasi berminggu-minggu. Lalu seseorang bilang: *"Coba deploy API-nya tanpa proteksi."*

Dalam hitungan jam, orang lain bisa query modelku dan merekonstruksi behavior-nya. **Model stealing.**

Sejak itu aku paham: model perlu diamankan, sama seperti kode dan database.

---

### Ancaman

**1. Model Stealing**

Seseorang bisa meniru model dengan query berulang.

Mitigasi: Rate limiting, batasi jumlah query per user, monitoring.

**2. Adversarial Attack**

Input sengaja dimanipulasi untuk menipu model.

Contoh: Stiker di rambu STOP bikin mobil otonom bacanya "60 km/jam".

Mitigasi: Adversarial training, input preprocessing.

**3. Data Extraction**

Attacker bisa mengekstrak data training dari model.

Mitigasi: Differential privacy, regularisasi.

---

### Praktik

- **Akses terbatas** — model cuma bisa diakses via API terautentikasi
- **Rate limiting** — batasi query per user
- **Monitoring** — deteksi pola mencurigakan
- **Enkripsi** — simpan model file di tempat aman

> *"Model yang tidak diamankan adalah aset berharga yang diletakkan di pinggir jalan."*
