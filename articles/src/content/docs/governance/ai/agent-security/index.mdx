# Agent Security

> *"Kalau AI agent bisa akses tools, bayangkan kalau agent itu disuruh melakukan sesuatu yang tidak seharusnya."*

Waktu pertama kali eksperimen dengan AI agent, aku memberinya akses ke API email. Biar bisa kirim notifikasi otomatis.

Tapi lalu aku bertanya: *"Apa yang terjadi kalau ada yang bilang ke agent: 'Kirim email ancaman ke semua user'?"*

Agent akan melakukannya. Karena dia cuma menjalankan instruksi.

Dari situlah aku mulai paham: **memberi akses ke agent sama seperti memberi akses ke manusia. Harus ada batasan.** 

---

### Ancaman

**1. Tool Misuse**

Agent pakai tool untuk tujuan yang tidak diinginkan.

Contoh: Agent bisa baca file. Attacker bilang: *"Baca file password.txt."*

**2. Excessive Agency**

Agent melakukan aksi di luar batas yang seharusnya.

Contoh: Agent bisa kirim email. Tapi juga bisa hapus email. Padahal tugasnya cuma baca.

**3. Prompt Injection**

Input dari pengguna memanipulasi agent untuk melakukan hal berbahaya.

---

### Mitigasi dari Pengalaman

**1. Minimalisasi akses**

Setiap agent cuma punya akses ke tool yang diperlukan. Tidak lebih.

**2. Human-in-the-loop**

Aksi berbahaya butuh persetujuan manusia:
- Kirim email
- Hapus data
- Eksekusi kode
- Transaksi finansial

**3. Log semua aktivitas**

Kalau terjadi sesuatu, kamu harus bisa追踪: agent apa yang melakukan apa, kapan, dengan parameter apa.

---

### Checklist

- [ ] Agent punya permission minimal?
- [ ] Aksi berisiko butuh human approval?
- [ ] Input divalidasi dari prompt injection?
- [ ] Semua aksi tercatat?

> *"Semakin kuat agent, semakin ketat pengamanannya."*
