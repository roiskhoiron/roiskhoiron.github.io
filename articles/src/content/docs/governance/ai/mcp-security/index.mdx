# MCP Security

> *"MCP kayak memberi AI remote control ke aplikasi. Pastikan remote control-nya tidak bisa dipakai sembarangan."*

Waktu pertama kali nyoba MCP (Model Context Protocol), aku kasih AI agent akses ke database — read-only, untuk menjawab pertanyaan.

Tapi lalu aku mikir: *"Apa yang terjadi kalau ada prompt injection? Agent bisa query apa aja?"*

Ternyata bisa.

---

### Ancaman

| Ancaman | Contoh |
|---------|--------|
| **Tool abuse** | Agent panggil tool berkali-kali sampai overload |
| **Data access** | Agent baca data yang tidak seharusnya |
| **Privilege escalation** | Agent dapet akses lebih dari yang seharusnya |

---

### Yang Kupelajari

**1. Scope permission**

Setiap koneksi MCP harus punya scope terbatas. Read-only vs read-write. Data apa yang bisa diakses.

Aku bagi:
- Agent customer support: bisa baca data user, tidak bisa edit
- Agent admin: bisa CRUD, tapi butuh approval untuk delete

**2. Rate limiting**

Batasi berapa kali agent bisa panggil tool per menit. Mencegah abuse.

**3. Audit trail**

Log setiap interaksi MCP. Siapa yang minta, tool apa yang dipanggil, apa hasilnya.

**4. Input validation**

Sama seperti input user, input dari agent ke tool harus divalidasi.

---

### Checklist

- [ ] Setiap tool punya scope?
- [ ] Ada rate limiting?
- [ ] Interaksi tercatat?
- [ ] Input divalidasi?

> *"Semakin banyak akses yang kamu beri ke agent, semakin ketat pengamanannya."*
