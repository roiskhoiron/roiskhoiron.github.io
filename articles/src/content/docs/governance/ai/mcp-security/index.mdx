# MCP Security

> *"MCP bikin agent bisa pake tools apa aja. Tapi kalau tools-nya tidak aman, agent jadi vector serangan."*

MCP (Model Context Protocol) adalah protokol yang memungkinkan AI agent berkomunikasi dengan tools. Keren — tapi setiap koneksi baru adalah potensi celah keamanan baru.

---

### Ancaman

**1. Tool spoofing**

Tools palsu yang kelihatan seperti tools resmi. Agent panggil tools palsu → data dikirim ke attacker.

**2. Data interception**

Komunikasi antara agent dan tools dicegat. Tanpa enkripsi, data yang dikirim bisa dibaca pihak ketiga.

**3. Unauthorized access**

Tools yang tidak dilindungi autentikasi. Siapa pun bisa panggil tools itu — termasuk agent jahat.

---

### Mitigasi

**Autentikasi dan otorisasi**

Setiap tools MCP harus punya autentikasi. Token, API key, atau OAuth. Tools tidak boleh bisa diakses tanpa kredensial.

**Enkripsi**

Komunikasi antara agent dan tools harus dienkripsi. HTTPS/TLS minimum.

**Rate limiting**

Batasi berapa kali agent bisa panggil tools dalam periode tertentu. Cegah abuse.

**Audit log**

Setiap panggilan tools tercatat: agent mana, tools apa, parameter apa, hasil apa. Untuk investigasi kalau ada insiden.

**Tool whitelist**

Agent cuma bisa akses tools yang sudah di-whitelist. Tidak bisa panggil tools sembarangan di internet.

---

### Yang kulakukan

Di CodingSkuy, tools MCP terbatas pada:
1. Search index artikel
2. Baca artikel tertentu
3. Generate ringkasan

Tidak ada akses ke database user, tidak ada akses ke environment variables, tidak ada akses delete.

> *"Keamanan MCP bukan tentang tools-nya — tapi tentang siapa dan apa yang bisa akses tools itu."*
