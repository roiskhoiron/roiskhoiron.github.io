# Naming Conventions

> *"Pernah debug 2 jam. Ternyata salah paham: `getPrice()` mengembalikan harga setelah diskon, bukan harga asli."*

Nama yang menyesatkan adalah bug yang tidak kelihatan. Kode berjalan normal. Tapi logic-nya salah karena programmer lain (atau diri sendiri) salah paham dengan nama fungsi.

---

### Kenapa Nama Penting?

Nama adalah dokumentasi paling murah. Nama yang baik:
- Mengurangi kebutuhan komentar
- Mempercepat onboarding engineer baru
- Mencegah misinterpretasi

---

### Aturan dari Pengalaman

**1. Eksplisit, bukan singkat**

| ❌ | ✅ |
|---|---|
| `getData()` | `getUserByEmail()` |
| `delUser()` | `deleteUserAccount()` |
| `proc()` | `processPayment()` |

**2. Satu nama untuk satu konsep**

Kalau sudah pakai `fetchUsers()`, jangan ada `getUserList()` di file lain untuk hal yang sama.

**3. Boolean harus jelas iya/tidak**

| ❌ | ✅ |
|---|---|
| `user` | `isLoggedIn` |
| `flag` | `hasPermission` |
| `status` | `isActive` |

### Convention per Bahasa

| Bahasa | Folder | File | Fungsi |
|--------|--------|------|--------|
| Flutter/Dart | snake_case | snake_case | camelCase |
| Java/Kotlin | snake_case | PascalCase | camelCase |
| PHP/Laravel | snake_case | snake_case | camelCase |
| JavaScript | kebab-case | kebab-case | camelCase |
| Python | snake_case | snake_case | snake_case |

**Konsisten lebih penting dari benar.**

---

### Tips

- Kalau bingung nama, minta review nama ke teman. Kalau mereka paham, nama sudah cukup baik
- Jangan takut rename. IDE bisa rename otomatis
- Gunakan istilah domain, bukan istilah teknis. `checkoutCart()` lebih baik dari `postCartData()`

> *"Nama yang baik adalah dokumentasi yang bisa jalan sendiri."*
