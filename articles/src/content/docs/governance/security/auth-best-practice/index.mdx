# Auth Best Practice

> *"Dulu aku simpan password di database tanpa hash. Untung aplikasinya cuma dipake 3 orang."*

Waktu belajar backend pertama kali — di SMK — aku bikin sistem login simpel. Password disimpan di database sebagai teks biasa (plain text). Aku tidak tahu itu berbahaya. Aku juga tidak tahu ada yang namanya hashing, salt, bcrypt.

Sejak itu, setiap urusan auth, aku selalu terapkan best practice.

---

### Hal Fundamental

**1. Jangan pernah simpan password**

Simpan hash, bukan password. Pakai bcrypt, argon2, atau PBKDF2. Jangan MD5 atau SHA — itu untuk checksum, bukan untuk password.

**2. Gunakan HTTPS**

Setiap request login/register harus HTTPS. Kalau HTTP, password dikirim dalam teks biasa — bisa dibaca siapa pun di jaringan yang sama.

**3. Rate limiting**

Batasi percobaan login. 5 kali salah → tunggu 5 menit. Atau pakai CAPTCHA. Mencegah brute force.

---

### JWT

JWT (JSON Web Token) populer untuk session management. Tapi ada hal yang perlu diperhatikan:

- **Jangan simpan rahasia di JWT** — JWT di-encode, bukan di-enkripsi. Siapa pun bisa decode dan lihat isinya. Hanya signature yang tidak bisa dipalsukan.
- **Set expiration** — JWT tanpa expiry sangat berbahaya. Kalau bocor, penyerang bisa akses selama-lamanya.
- **Refresh token** — access token pendek (15 menit), refresh token lebih panjang (7 hari). Kalau access token bocor, dampak terbatas.

### Session Management

- **Logout harus invalidate session** — jangan cuma hapus token di client. Server juga harus invalidate.
- **Multi-device** — apakah user bisa login di banyak device? Kalau iya, butuh session management per device.
- **Deteksi anomali** — login dari IP berbeda dalam waktu singkat? Notifikasi user.

---

### Praktik yang kulakukan

- Firebase Auth untuk CodingSkuy — tidak perlu handle password sendiri
- Rate limiting di API — 5 attempts per menit per IP
- Logout di semua device kalau user ganti password

> *"Auth adalah gerbang pertama. Kalau gerbangnya bocor, sisanya tidak penting."*
