# Data Protection

> *"Log error yang kupikir aman — ternyata menyimpan password user dalam bentuk teks biasa. Beruntung ketahuan sebelum bocor."*

Waktu debugging, aku log request body untuk lihat apa yang dikirim user. Lupa filter field sensitif. Di log tercatat: email, nama, dan password. Log itu bisa diakses tim lain.

Sejak itu, semua logging harus difilter. Data sensitif tidak boleh tercatat.

---

### Data Sensitif

Yang termasuk data sensitif:
- Password, token, API key
- Data pribadi (nama, email, alamat, nomor telepon)
- Data finansial (nomor kartu, rekening)
- Data kesehatan

---

### Prinsip

**1. Data minimization**

Kumpulkan data yang diperlukan saja. Jangan minta nomor telepon kalau hanya perlu email.

Ini bukan cuma soal keamanan — tapi juga UX. Makin sedikit data yang diminta, makin kecil risiko.

**2. Enkripsi**

- **In transit** — HTTPS/TLS. Semua data yang dikirim harus dienkripsi.
- **At rest** — data di database dienkripsi. Kalau database bocor, data tidak bisa dibaca.

**3. Akses terbatas**

Tidak semua orang perlu akses ke semua data. Developer frontend tidak perlu akses ke database production.

Gunakan prinsip least privilege. Setiap role cuma punya akses ke data yang diperlukan untuk tugasnya.

**4. Audit log**

Siapa yang akses data apa dan kapan? Catat. Untuk investigasi kalau ada insiden.

---

### Praktik

- **Jangan log password** — filter field sensitif di logging
- **Mask data sensitif di UI** — nomor kartu: **** **** **** 1234
- **Hapus data yang tidak diperlukan** — kalau user minta hapus akun, hapus datanya
- **Backup rutin** — dan backup harus dienkripsi juga

---

### Response Plan

Kalau terjadi kebocoran:
1. **Identifikasi** — data apa yang bocor? Seberapa parah?
2. **Stop the leak** — tutup celah. Matikan akses yang tidak perlu.
3. **Notifikasi** — beri tahu user yang terdampak.
4. **Evaluasi** — kenapa ini terjadi? Sistem apa yang perlu diperbaiki?

> *"Data protection bukan cuma teknologi — tapi proses. Mulai dari bagaimana data dikumpulkan sampai bagaimana data dihapus."*
