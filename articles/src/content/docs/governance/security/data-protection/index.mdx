# Data Protection

> *"Data adalah aset paling berharga. Jaga seperti kamu menjaga milikmu sendiri."*

---

### Klasifikasi Data

| Level | Contoh | Proteksi |
|-------|--------|----------|
| **Public** | Nama produk, halaman blog | Standar |
| **Internal** | Kode sumber, dokumentasi | Akses terbatas |
| **Confidential** | Data pengguna, financial | Enkripsi + akses ketat |
| **Restricted** | Password, token, Kunci API | Enkripsi kuat + audit |

---

### Enkripsi

**In Transit:**
- Wajib HTTPS (TLS 1.2+)
- HSTS header
- Certificate pinning untuk mobile app

**At Rest:**
- Enkripsi database (AES-256)
- Enkripsi file upload
- Jangan simpan sensitive data di log

### Praktik

- **Minimalisasi** — hanya kumpulkan data yang benar-benar dibutuhkan
- **Anonimisasi** — hapus PII dari data analytics
- **Retensi** — tentukan berapa lama data disimpan, hapus setelah tidak diperlukan
- **Backup** — backup terenkripsi dengan disaster recovery plan

### Regulasi

Pahami regulasi yang relevan:
- **UU PDP** (Indonesia) — wajib untuk aplikasi dengan pengguna Indonesia
- **GDPR** — jika ada pengguna Eropa
- **HIPAA** — jika berurusan dengan data kesehatan (AS)
