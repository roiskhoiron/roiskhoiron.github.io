# Threat Modeling

> *"Aku pernah pikir: 'Ah, nggak mungkin ada yang mau retas aplikasi kecilku.' Sampai data temanku bocor lewat celah yang tidak pernah kupikirkan."*

Threat modeling adalah proses mengidentifikasi potensi ancaman sebelum terjadi. Bukan paranoid — tapi persiapan.

---

### STRIDE Framework

Microsoft punya STRIDE — framework untuk kategorisasi ancaman:

**Spoofing** — penyamar. Attacker berpura-pura jadi user lain.
Mitigasi: autentikasi kuat, jangan simpan password.

**Tampering** — manipulasi data. Attacker mengubah data di transit.
Mitigasi: enkripsi, checksum, validasi input.

**Repudiation** — penyangkalan. User melakukan sesuatu lalu menyangkal.
Mitigasi: audit log, signature.

**Information Disclosure** — kebocoran informasi. Data sensitif terbaca pihak tidak berhak.
Mitigasi: enkripsi, akses kontrol, minimal data exposure.

**Denial of Service** — layanan tidak bisa diakses.
Mitigasi: rate limiting, auto-scaling, resource management.

**Elevation of Privilege** — user biasa bisa akses fitur admin.
Mitigasi: otorisasi ketat, least privilege.

---

### Cara Melakukan Threat Modeling

**1. Diagram arsitektur**

Gambar: client → API → database. Tandai setiap titik di mana data mengalir.

**2. Identifikasi ancaman per titik**

Setiap titik punya ancaman. Client bisa diretas. API bisa kena injection. Database bisa bocor.

Untuk setiap ancaman, tanya:
- Seberapa mungkin terjadi?
- Kalau terjadi, seberapa parah?

**3. Prioritaskan**

Fokus ke ancaman dengan probability tinggi dan impact tinggi.

**4. Rencanakan mitigasi**

Untuk setiap ancaman prioritas:
- Kurangi probability — "pasang rate limiting"
- Kurangi impact — "matikan database connection kalau anomali"

---

### Contoh Sederhana

Aplikasi CodingSkuy:
- **Client (mobile app)** — ancaman: reverse engineering → mitigasi: obfuscation
- **API (Firebase Functions)** — ancaman: injection → mitigasi: validasi input
- **Database (Firestore)** — ancaman: unauthorized access → mitigasi: security rules

---

### Tools

- **Threat Dragon** — OWASP, gratis, open source
- **Microsoft Threat Modeling Tool** — untuk Windows, komprehensif
- Cukup pake whiteboard untuk proyek kecil

> *"Threat modeling bukan untuk menghilangkan semua risiko — tapi untuk tahu risiko apa yang kamu hadapi dan bagaimana menghadapinya."*
