# Security Review

> *"Aku pikir aplikasi sudah aman. Tapi setelah security review, nemu 10 celah. Termasuk yang kritis."*

Security review berbeda dengan code review. Code review fokus ke logika, maintainability. Security review fokus ke: bagaimana aplikasi bisa diretas.

Aku pernah minta teman audit aplikasi. Dalam 30 menit, dia nemu:
- API endpoint tanpa autentikasi
- Input tidak divalidasi
- Error message bocor informasi server
- Token tidak expired

Itu wake-up call.

---

### Area yang Dicek

**1. Authentication & Session**

- Apakah password di-hash dengan algoritma kuat? (bcrypt/argon2)
- Apakah session token aman? (HttpOnly, Secure, SameSite)
- Apakah ada rate limiting untuk login?
- Apakah logout beneran invalidate session?

**2. Authorization**

- Apakah setiap endpoint cek otorisasi?
- Apakah ada IDOR (Insecure Direct Object Reference)?
- Apakah admin endpoint beneran cuma bisa diakses admin?

**3. Input Validation**

- Apakah input divalidasi di server? (jangan cuma di client)
- Apakah ada SQL injection risk?
- Apakah ada XSS (Cross-Site Scripting) risk?
- File upload — apakah tipe file dan ukuran dibatasi?

**4. Data Exposure**

- Apakah data sensitif di-log?
- Apakah error response bocor informasi internal? (stack trace, DB detail)
- Apakah API response mengandung field yang tidak perlu?

**5. Configuration**

- Apakah environment variables aman? (tidak di commit)
- Apakah CORS dikonfigurasi dengan benar? (jangan allow all)
- Apakah dependency up-to-date? (cek CVE)

---

### Cara Melakukan

**Manual review** — baca kode dengan perspektif security. Fokus ke area yang disebut di atas. Bisa butuh 1-4 jam tergantung kompleksitas.

**Automated scanning** — pake tools:
- `npm audit` / `flutter pub audit` — cek dependency
- OWASP ZAP — automated security scanner
- `npx snyk test` — cek celah keamanan

**Penetration testing** — simulasi serangan. Coba exploit celah yang mungkin ada. Bisa dilakukan internal atau hire pihak ketiga.

---

### Frekuensi

- Setiap kali ada fitur yang handle data sensitif
- Sebelum rilis major
- Setiap 6 bulan untuk existing system

> *"Security review bukan one-time activity. Ancaman berubah, dan sistem harus selalu dievaluasi."*
